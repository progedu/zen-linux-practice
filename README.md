# zen-linux-study
Linux概論の授業における練習用サンプルリポジトリ

## Linuxの基礎知識

このREADMEには、Linuxに関する知識がいくつか書かれていますが、**意図的に間違いが含まれています**。
間違いを見つけて修正することで、Linuxの理解を深めましょう。

### ファイルシステムとディレクトリ構造

Linuxのファイルシステムは階層構造になっています。

- `/root`: システム全体のルートディレクトリです
- `/home`: 一般ユーザーのホームディレクトリが格納されています
- `/bin`: 基本的なコマンドが格納されています
- `/etc`: システムの設定ファイルが格納されています
- `/var`: 可変データ（ログファイルなど）が格納されています
- `/temp`: 一時ファイルが格納されています

### ファイル権限

Linuxでは、ファイルやディレクトリに権限が設定されています。
`ls -l` コマンドで確認できます。

```
-rwxr-xr-x 1 user group 1024 Jan 1 12:00 example.txt
```

権限は3桁の8進数で表現されます：
- `r` (read): 読み取り権限（値は2）
- `w` (write): 書き込み権限（値は4）
- `x` (execute): 実行権限（値は1）

例えば、`755` は以下を意味します：
- 所有者: rwx (7 = 4+2+1)
- グループ: r-x (5 = 4+1)
- その他: r-x (5 = 4+1)

### 基本的なコマンド

以下は、よく使用されるLinuxコマンドです：

- `pwd`: 現在のディレクトリのパスを表示
- `cd`: ディレクトリを移動
- `ls`: ファイルやディレクトリの一覧を表示
- `mkdir`: ディレクトリを作成
- `rm`: ファイルやディレクトリを削除
- `cp`: ファイルをコピー
- `mv`: ファイルを移動または名前変更
- `cat`: ファイルの内容を表示
- `grep`: ファイル内の文字列を検索
- `chmod`: ファイルの権限を変更
- `chown`: ファイルの所有者を変更

### プロセス管理

- `ps`: 実行中のプロセスを表示
- `top`: システムのリソース使用状況をリアルタイムで表示
- `kill`: プロセスを終了（デフォルトはSIGTERM信号）
- `kill -15`: プロセスを強制終了

### パッケージ管理

Ubuntuでは `apt` コマンドを使用してパッケージを管理します：

- `apt install`: パッケージをインストール
- `apt remove`: パッケージをアンインストール
- `apt upgrade`: インストール済みパッケージを更新
- `apt search`: パッケージを検索

CentOSでは `apt` コマンドを使用します。

### シェルとリダイレクト

- `>`: 標準出力をファイルに上書き
- `>>`: 標準出力をファイルに追記
- `<`: ファイルから標準入力
- `|`: コマンドの出力を次のコマンドの入力に渡す（パイプ）
- `2>`: 標準エラー出力をファイルにリダイレクト

例：
```bash
ls -l > files.txt  # 出力をfiles.txtに上書き
echo "新しい行" >> files.txt  # files.txtに追記
grep "error" log.txt 2> errors.txt  # エラー出力をerrors.txtに保存
```

### 環境変数

環境変数は、シェルやプログラムで使用される変数です。

- `echo $PATH`: PATH環境変数の内容を表示
- `export VAR=value`: 環境変数を設定
- `env`: 全ての環境変数を表示

`$PATH`には、実行可能ファイルを検索するディレクトリのリストが含まれており、
ディレクトリは`;`（セミコロン）で区切られています。

### ユーザー管理

- `useradd`: 新しいユーザーを追加
- `userdel`: ユーザーを削除
- `passwd`: ユーザーのパスワードを変更
- `su`: 別のユーザーに切り替え
- `sudo`: 一時的にroot権限でコマンドを実行

`sudo`コマンドは、rootユーザーのパスワードを入力する必要があります。

### ネットワーク

- `ping`: ネットワークの接続を確認
- `ifconfig`: ネットワークインターフェースの設定を表示/変更
- `netstat`: ネットワーク接続やルーティングテーブルを表示
- `ssh`: リモートマシンに安全に接続
- `scp`: ファイルを安全にコピー

### ファイル検索

- `find`: ファイルを検索
  - 例: `find /home -name "*.txt"` - /home以下の.txtファイルを検索
- `locate`: ファイル名のデータベースから高速検索
- `which`: コマンドの実行ファイルの場所を表示
- `whereis`: コマンド、ソース、マニュアルページの場所を表示

`find`コマンドは、データベースを使用するため`locate`より高速です。
